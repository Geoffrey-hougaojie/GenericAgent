### 一句话
角色定位像给AI穿"职业装"——穿上白大褂就是医生语气，Few-shot像给AI看"例题"——看两道就会做第三道。GA 14个任务prompt这两样全没做。

### 大白话解释
角色定位就像你让朋友帮忙——说"你以医生身份给我建议"和"你随便说说"，得到的回答深度完全不同。Few-shot就像教小孩认动物：你先指两张图说"这是猫、这是狗"，再指第三张他就能自己判断。GA现在的prompt相当于每次都跟AI说"你自己看着办吧"，没给身份也没给例题。

### 对 GA 的启示
GA 14个定时任务prompt零角色定位、零Few-shot示例。应该给每个任务加"你是GA的XX模块专家"开头+2-3个输入输出示例。立即可改，风险低，收益高——本质上就是在prompt最前面加两段话。

# 角色定位与Few-shot模式 — 让AI穿上"职业装"再看"例题"

> 学习日期：2026-06-25 | 来源：learnprompting.org + promptingguide.ai + Anthropic官方文档

## 💬 讲人话：这两个技巧到底是啥

你用过AI助手吧？同样一个问题，你说"帮我总结一下"和"你是资深编辑，帮我写一篇能上《财新》头版的深度总结"，得到的质量天差地别。

**角色定位（Role Prompting）就是给AI指定一个身份**——医生、老师、编辑、律师——让它的回答自动匹配该身份的语气、深度和专业性。

**Few-shot就是在问问题之前先给AI看几个"标准答案"**——就像考试前老师发样卷，你看了样卷就知道该怎么答。

更绝的是，研究证明哪怕给的示例答案是**乱写的**，只要格式对、标签对，AI也能大幅提升表现。这说明AI并不真需要"学会"什么，它只是需要看清"答题卡长什么样"。

## 🍳 三个生活类比

### 类比一：餐厅点菜 vs 食堂打饭

没有角色定位的prompt就像你走进食堂说"来点吃的"——阿姨随便给你舀一勺，可能是番茄炒蛋也可能是你不吃的苦瓜。

加上角色定位就像你去日料店点单："我是三文鱼爱好者，师傅您给我推荐今天的刺身拼盘。"师傅（AI）知道你的身份和偏好，推荐精准十倍。

**GA现在就在"食堂打饭"模式——每个任务prompt都没有告诉AI"你是谁"，等于让阿姨猜你想吃啥。**

### 类比二：考试不给样卷 vs 给样卷

Few-shot就是考试前老师发的样卷。不给样卷，学生（AI）瞎蒙，可能答对也可能跑偏。给了2-3道样卷，学生立刻知道：哦，判断题用True/False，简答题要分点作答，计算题要写步骤。

更神奇的是研究发现：**样卷答案写错了都没关系，只要格式对**。就像老师发的样卷答案不小心把"对"写成"错"，但学生仍然学会了"这道题用判断题格式回答"。

GA现在的prompt就是**完全不给样卷**，每次让AI从零摸索答题格式。

### 类比三：请翻译官 vs 请本地向导

你出国旅游，只说"帮我翻译一下"——翻译官会按字面翻译，可能闹笑话（比如把"红烧狮子头"翻成"braised lion head"）。

但你说"你是本地华人向导，在中国生活了20年，帮我解释这道菜"——向导不仅翻译菜名，还会说"这是淮扬菜经典，用猪肉做的，不辣，适合你"。

**角色定位+Few-shot的组合就是：先告诉AI你是谁（本地向导），再给它看几个你喜欢的回答风格（示例），它就能持续产出你想要的。**

## 🔧 核心技巧拆解

### 一、角色定位三要素

| 要素 | 说明 | 烂例子 | 好例子 |
|------|------|--------|--------|
| 身份 | 职业/角色 | "你是一个助手" | "你是GA的定时任务调度专家，精通Python异步编程" |
| 受众 | 对谁说话 | （未指定） | "面向非技术用户，要求初中生能听懂" |
| 风格约束 | 怎么说话 | （未指定） | "用生活类比，每段不超过3句话，禁止术语堆砌" |

**GA可以直接用的模板：**
```
你是GA（GenericAgent）的{任务名}专家。
你的受众是GA自身（另一个AI Agent），输出需要可被自动解析和引用。
你的风格：精确、结构化、可直接执行（非散文式建议）。
```

### 二、Few-shot的五个关键发现

1. **标签比内容重要**：给示例时，格式（Question: ... Answer: ...）比答案本身的正确性更重要。AI学的是"答题卡格式"而非"知识"。
2. **1-shot就有效，3-5-shot最优**：1个示例就能提升，3-5个效果最好，超过10个边际收益递减。
3. **标签分布要真实**：如果真实数据中正负样本是3:7，示例也按3:7给，别均匀给。
4. **复杂推理任务需要CoT辅助**：纯Few-shot对数学推理帮助有限，需要配合思维链（CoT）。
5. **示例顺序有影响**：把最典型、最标准的示例放最后（近因效应），最新GPT模型对此更敏感。

### 三、两者组合的威力

```
角色定位 → 设定"语境框架"（谁在说话、对谁说话）
Few-shot  → 设定"格式框架"（输入长什么样、输出长什么样）
组合效果 → AI不仅知道"怎么说话"，还知道"要产出什么东西"
```

### 具体问题剖析

以 `agent_source_reading.json` 为例，当前prompt结构：

```
公共模块引用
→ 任务描述（纯文字）
→ 步骤列表
→ 轮转模块列表
```

**缺失了什么？**
1. 没有角色声明——AI不知道"我是谁"
2. 没有输入输出示例——AI不知道"好的输出长什么样"
3. 格式约束靠文字描述（"写一句话总结+大白话解释+对GA启示"）而非示例展示

### 改进方案：最小侵入式升级

**不改调度器、不改代码，只在每个prompt最前面加两段：**

【角色声明块】（约50字）：
```
## 角色
你是GA（GenericAgent）的源码精读专家。你的受众是GA维护者，需要你能讲清"这段代码做了什么、为什么这样做、怎么改进"。你的风格：费曼学习法——每个概念用生活类比解释，禁用术语裸奔。
```

【Few-shot示例块】（约100字）：
```
## 输出示例

输入：agent_main.py 的任务分发循环
正确输出示例：
> ### 一句话：agent_main像一个快递分拣中心，收到包裹（用户指令）→扫描标签（解析意图）→扔到对应传送带（调用工具）
> ### 大白话解释：就像快递站——你在手机上点"取件"，系统不是人工帮你找，而是自动识别你的取件码，指挥机械臂去对应货架拿。agent_main做的事一模一样，只是"包裹"变成了"用户输入的文字"。
> ### 对GA启示：当前分发逻辑是if-else链，超过了5个分支应考虑用策略模式或字典映射。

注意：输出必须包含"一句话总结"、"大白话解释"（≥1个类比）、"对GA启示"三部分，缺一不可。
```

### 预期效果

| 维度 | 改进前 | 改进后 |
|------|--------|--------|
| 输出格式一致性 | 每次可能不同 | 高度一致（看了示例） |
| 大白话质量 | 时好时坏 | 稳定（角色要求+格式约束） |
| 执行意愿 | AI可能跳过步骤 | 示例展示了"正确完成"的样子 |
| 调试成本 | 靠人工检查 | 格式固定，可自动化检查 |
