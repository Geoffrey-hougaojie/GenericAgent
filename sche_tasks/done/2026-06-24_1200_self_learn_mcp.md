# MCP Tools 自学报告 — 2026-06-24 12:00

## 普通用户阅读入口
> 如果同事问"AI 整天学的这些词到底什么意思"，直接看下面这部分。

MCP Tools 就是 AI 的 **"遥控器按钮清单"**。比如说你家的电视遥控器上有"音量+"、"换台"、"电源"，每个按钮都有名字和用法——MCP Tools 就是给 AI 配的遥控器：AI 先看一眼遥控器上有什么按钮（tools/list），然后按想要的按钮（tools/call，比如"音量+5"），电视就执行了（result）。

更具体一点：
- **工具定义** = 按钮的说明书：名字叫什么、干什么用的、需要什么参数
- **列出工具** = AI 扫一眼遥控器面板
- **调用工具** = AI 按下按钮
- **返回结果** = 电视屏幕上显示的反馈

就这么简单。剩下那些 terms（JSON-RPC、inputSchema、handle）都只是"按钮说明书用什么格式写"的实现细节。

# 一句话摘要
MCP Tools 就是 AI 的工具箱目录——每个工具有名字、说明书和参数表，AI 看一眼就知道怎么用。

## 大白话解释
MCP Tools 就像餐厅菜单：AI（顾客）先要菜单看看有什么菜（tools/list），挑一个想要的告诉服务员（tools/call，带上"少辣"这类参数），厨房做好端上来（result）。如果菜做错了，服务员会告诉你错在哪，你可以改要求重来。工具定义就是菜品的"名字+描述+需要什么食材"，全部用 JSON 格式写清楚，AI 读完就懂。

## 对 GA / 工作的启示
GA 的 code_run、file_read 等本身就是 MCP Tool 概念的具体实现——只是没用标准格式。如果按 MCP 的 inputSchema（JSON Schema）标准化每个工具的参数字段，LLM 对参数格式的幻觉会大幅减少。FastMCP 的 @mcp.tool() 装饰器模式可以直接参考来重构 GA 的工具注册机制。

### 核心知识点

1. **Tool 定义** = `{name, description, inputSchema}` 三元组，inputSchema 用 JSON Schema 描述参数
2. **tools/list** — 客户端发现可用工具，支持分页（cursor）+ 缓存（ttlMs）+ 变化通知（listChanged）
3. **tools/call** — 调用工具，传 name + arguments，返回 result（content 数组 + isError 标记）
4. **两种错误** — Protocol Error（工具不存在等，JSON-RPC error）vs Tool Execution Error（参数不对，isError: true，LLM 可自修复）
5. **有状态工具** — 通过 handle（不透明 ID）模拟跨调用状态，类似银行取号
6. **outputSchema** — 可选，定义工具输出的结构化格式，方便程序解析
