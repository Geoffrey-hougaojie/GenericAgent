## 关键问题与修复

### 🔴 Bug: JS `atob()` → JSON.stringify 双重 UTF-8 编码
**现象**: 含中文/非ASCII字符的 24 个文件 SHA 校验全部失败。
**根因**: JS `atob()` 将 base64 解码为二进制字符串(char codes 0-255)，随后 `JSON.stringify` 序列化时对 >=128 的 code point 进行 UTF-8 编码，导致多字节展开（如 `\xe2` 变成 `\xc3\xa2`）。
**修复**: 改用 **raw base64 直传**，在 Python 端 `base64.b64decode()` 解码。对已双编码的历史数据，使用 `reverse_double_utf8()` 函数逆向还原。
**建议**: 更新 SOP 中的 JS fetch 模板，避免使用 atob() 处理 base64。
