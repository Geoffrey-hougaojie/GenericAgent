# MCP Python SDK 实战自学报告 — 2026-06-25 12:00

## 普通用户阅读入口
> 如果同事问"AI 整天学的这些词到底什么意思"，直接看下面这部分。

**MCP Server 就是给 AI 配的「万能遥控器制造工厂」。** 你家遥控器只能控制电视，但 MCP Server 可以让 AI 有一个遥控器，上面有「查天气」「算数学」「读文件」等按钮——想让 AI 干什么，就往遥控器上加什么按钮。

**Python SDK 就是造遥控器的工具箱。** 里面有个叫 FastMCP 的傻瓜工具：写一个普通 Python 函数，加 `@mcp.tool()` 这行标签，函数就自动变成 AI 遥控器上的按钮。三种传送方式：红外线（stdio，本机用）、WiFi（HTTP，远程用）、蓝牙（SSE，已淘汰）。

就这么简单。剩下那些 transport、Pydantic、CallToolResult 什么的，只是「遥控器用什么材料做、信号怎么编码」的实现细节。

---

### 一句话
FastMCP 装饰器让写 MCP Server 像贴标签一样简单——Python 函数加 @mcp.tool() 秒变 AI 可调用的工具。

### 大白话解释
就像开饭店加盟：你只负责做菜（写函数），总部帮你搞定菜单、服务员、外卖平台。FastMCP 就是 MCP Server 的加盟总部——函数名是菜名，docstring 是菜品描述，类型注解自动生成「需要什么食材」的清单。stdio 像堂食（本机直接通信），HTTP 像外卖（跨网络调用）。

### 对 GA 的启示
GA 当前的工具注册是手写 dict 配置，FastMCP 的装饰器 + 类型推导自动生成 JSON Schema 可直接借鉴——降低 LLM 参数幻觉。`json_response=True` 的结构化输出模式也值得引入 GA 的 code_run 返回值设计。

### 核心知识点

1. **FastMCP 三步创建 Server**：`FastMCP("name")` → `@mcp.tool()` 装饰器 → `mcp.run()`
2. **三大装饰器**：`@mcp.tool()` 动作、`@mcp.resource("uri://{p}")` 数据、`@mcp.prompt()` 模板
3. **三种 Transport**：stdio（本地进程）、Streamable HTTP（远程）、SSE（逐步淘汰）
4. **结构化输出**：返回 Pydantic BaseModel 自动生成 outputSchema，AI 解析更准
5. **参数自动推导**：类型注解 → JSON Schema inputSchema，零额外配置
6. **上下文获取**：`mcp.get_context()` 获取 session/client_id/request_id
7. **Low-Level Server**：毛坯房模式，完全自定义但需手写 handler
8. **v1.x 稳定 / v2 alpha**：v2 目标 2026-07-27 稳定，当前用 `mcp>=1.27,<2` 锁定
